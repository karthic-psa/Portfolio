Evaluation of Cloud based deep learning across GCP, AWS & Microsoft Azure


Decription:


Among the providers of public cloud there are three dominant players; Google Cloud Platform, Amazon Web Services and Microsoft Azure. Each of these service providers slightly differ from each other and directly compete on price. Knowing the advantages and disadvantages of each service can be a key factor for a business transitioning to cloud. However, because of their large scale and variety it is difficult to effectively compare what services are better suited to a given requirement. In our project, we compare different aspects of a machine learning service across all three platforms. We build classification models over datasets which vary in size and structure and compare ML results and evaluation metrics across three platforms. It is required to have prior knowledge of machine learning which will include tasks like acquiring relevant datasets, cleaning, and formatting data, splitting the dataset for evaluation (n-fold cross validation, percentage split), building models over dataset (some service providers have a black box to build models) and understanding evaluation metrics. A black box may be one or more custom algorithms working together to generate a general model from data. Collected data sets are cleaned and formatted to match the requirements of the platform. Store the data in respective cloud storage service and initiate machine learning instance. We then define formats for each data column and tell the CSP (cloud service provider) what is your target class (the part of the data set which has to be predicted). Once we confirm the structure of the data we configure the CSP to learn a model from the data. These configuration settings have to be recorded to understand how metrics vary across different configuration settings for the same CSP. The outputs are recorded to CSP’s storage service and are compared. Comparison between different settings will provide us enough information to analyze and select best CSP for a business.

Hybrid Genetic Algorithm Approach for Mobile Robot Path Planning

Decription:


The proposed hybrid approach integrates the trajectory planning with the path planning for the mobile robot navigation in the indoor simulated environment. GA (Genetic Algorithm) makes the robot to choose the optimal path selection to reach the goal with global path planning. GJK distance algorithm supports the collision avoidance of convex shaped dynamic obstacles during navigation. The GJK algorithm finds the obstacles position in the environment and incorporates the value in the fitness function during every step movement of the robot and adds the internal mechanism of obstacles values into the Genetic Algorithm which makes the effective optimal path selection in the environment. To ensure statistical significance of the hybrid approach, paired sample„t‟ test were performed using SPSS tool. Hybrid GA takes less time over GA in various tests of different shapes of obstacles in the environment. The overall various shaped obstacles the average time taken to reach the goal by Hybrid GA (164.33 seconds) is lesser than the GA (671.33 seconds) in reaching the goal. Integration of the distance based algorithm with heuristic approach makes the best optimal path selection with the effective obstacle prediction and avoidance in the environment.
